# Форматы данных

## Сервис получения событий

Сервис для получения событий из внешних систем

### Входные данные

Входными данными для сервиса получения событий являются события из внешней системы.  
События отправляются по HTTP в формате JSON. Для каждой внешней системы будет отдельный endpoint.

**Формат запроса**

POST /sources/{source_name}/events

```json
{
  "id": "evt_01H...", // id события
  "type": "order.created", // Тип события
  "created_at": "2026-04-11T10:15:30Z", // ISO-8601
  // Пейлоад события
  "data": {
    "order_id": "123",
    "amount": 1990,
    "currency": "RUB"
  }
}
```

### Выходные данные

Сервис получения публикует сообщения с данными события в kafka. Сообщения представляют собой JSONы

**Топик: routing.requests**

```json
{
  "event": {
    "id": "evt_01H...",
    "source": "crm-1",
    "type": "order.created",

    // Пейлоад
    "data": {
      "order_id": "123",
      "amount": 1990,
      "currency": "RUB"
    }
  },

  "ingested_at": "2026-04-11T10:30:30Z", // ISO-8601 Время получения события
  "trace_id": "trace-id"
}
```

## Сервис подписок

Сервис для получения соответствия событию адерса вебхука

_11.04.2026 Tip_
Пока что описываем только получение адреса вебхука. Заведение подписки будет сделано в третьем спринте

### Входные данные

Входные данные сервиса подписок совпадают с выходными данными сервиса получения событий. Данные беруться из Kafka

**Топик: routing.requests**

```json
{
  "event": {
    "id": "evt_01H...",
    "source": "crm-1",
    "type": "order.created",

    // Пейлоад
    "data": {
      "order_id": "123",
      "amount": 1990,
      "currency": "RUB"
    }
  },

  "ingested_at": "2026-04-11T10:30:30Z", // ISO-8601 Время получения события
  "trace_id": "trace-id"
}
```

### Выходные данные

Выходные данные сервис подписок представляют собой JSON с информацией об адресе вебхука. Данные кладуться в kafka

**Топик: deliveries.to_send**

```json
{
  "delivery_id": "dlv_01H...", // id доставки
  "event": {
    "id": "evt_01H...",
    "data": {
      "order_id": "123",
      "amount": 1990,
      "currency": "RUB"
    }
  },

  "subscription": {
    "id": "sub_01H...", // id подписки в сервисе подписок
    "destination_url": "https://example.com/webhooks/orders",
    "method": "POST",
    "headers": {
      "Content-Type": "application/json"
    }
  },

  "mapped_at": "2026-04-11T10:30:30Z", // ISO-8601 Время создания задачи на доставку
  "trace_id": "trace-id" // trace-id из сервиса получения событий
}
```

## Сервис доставки

Выполняет доставку вебхуков, имеет защиту от деградации получателя

### Входные данные

Входные данные сервиса доставки совпадают с выходными данными сервиса подписок. Данные беруться из Kafka

**Топик: deliveries.to_send**

```json
{
  "delivery_id": "dlv_01H...", // id доставки
  "event": {
    "id": "evt_01H...",
    "data": {
      "order_id": "123",
      "amount": 1990,
      "currency": "RUB"
    }
  },

  "subscription": {
    "id": "sub_01H...", // id подписки в сервисе подписок
    "destination_url": "https://example.com/webhooks/orders",
    "method": "POST",
    "headers": {
      "Content-Type": "application/json"
    }
  },

  "mapped_at": "2026-04-11T10:30:30Z", // ISO-8601 Время создания задачи на доставку
  "trace_id": "trace-id" // trace-id из сервиса получения событий
}
```

### Выходные данные

В качестве выходных данных сервис доставки выполняет HTTP запрос:

- Метод HTTP запроса получается из поля "subscription.method"
- url запроса получается из поля "subscription.destination_url"
- Запрос содержит заголовки из поля "subscription.headers"
- Запрос передает данные из поля "event.data"
  - В случае POST запроса данные передаются в теле в формате JSON
  - В случае GET запроса данные передаются в квери-параметрах
